# AI 赋能周报导出模块 — 技术文档

## 1. 模块概述

本模块实现"一键导出 AI 赋能周报 Word 文档"功能。用户点击按钮后，系统加载 `.docx` 模板文件，将数据填入模板占位符，生成并下载完整的周报文件。

### 核心思路

**不使用 docxtemplater 等模板引擎**，而是直接操作 `.docx` 内部的 XML 文本（`word/document.xml`），用正则替换占位符。这样做的原因是 docxtemplater 对模板的 XML 结构有严格要求，而 Word 保存模板时经常拆分 XML 节点，导致引擎解析失败。

## 4. 数据模型

定义在 `src/data/mockWeeklyReport.ts`：

```typescript
export interface AlgorithmRow {
  seq: number        // 序号
  type: string       // 算法类型名称
  cameras: number    // 摄像头路数
  events: number     // 告警事件数
  accuracy: string   // 识别准确率（如 "100%"）
  remark: string     // 备注
}

export interface WeeklyReportData {
  // ── 基本信息 ──
  reportPeriod: string    // 报告周期，如 "2026年6月9日-2026年6月15日"
  reportUnit: string      // 填报单位
  reportDate: string      // 填报日期

  // ── 一、总体概况 ──
  newCameras: number      // 本周新增摄像头
  totalCameras: number    // 累计摄像头总数
  totalAlerts: number     // 累计告警事件总数
  accuracy: string        // 总体准确率

  // ── 二、算法明细表 ──
  algorithms: AlgorithmRow[]   // 算法明细行（表格循环数据）

  // ── 三、告警事件分析 ──
  distributionSummary: string  // 事件分布概述
  keyEventsSummary: string     // 重点事件说明

  // ── 四、准确率分析 ──
  overallAccuracy: string      // 整体准确率数值
  accuracyDirection: string    // 变化方向（"上升"/"下降"）
  accuracyChange: string       // 变化幅度
  falsePositives: number       // 误报数
  falseNegatives: number       // 漏报数
  optimizationMeasures: string // 优化措施

  // ── 五、问题与计划 ──
  problems: string             // 存在问题（支持换行）
  nextWeekPlan: string         // 下周计划
}
```

### 5.2 循环表格占位符

算法明细表使用 `{#algorithms}...{/algorithms}` 循环语法，横跨表格的 6 列：

| 列 | 占位符 | 数据来源 |
|----|--------|----------|
| 序号 | `{seq}` | `algo.seq` |
| 算法类型 | `{type}` | `algo.type` |
| 摄像头路数 | `{cameras}` | `algo.cameras` |
| 告警事件数 | `{events}` | `algo.events` |
| 识别准确率 | `{accuracy}` | `algo.accuracy` |
| 备注 | `{remark}` | `algo.remark` |

> **重要**：`{accuracy}` 同时存在于循环内（每个算法的准确率）和循环外（总体准确率）。代码必须**先展开循环再做简单替换**，否则循环内的 `{accuracy}` 会被提前替换为总体值。

## 6. 核心流程

```
用户点击按钮
    │
    ▼
ExportReportButton.vue
    │  @click="exportReport()"
    ▼
useWeeklyReport().exportReport(data?)
    │
    ├─ 1. 准备数据：data ?? generateMockReport()
    │
    ├─ 2. fetch('/templates/AI赋能周报模板.docx')
    │     └─ 获取 ArrayBuffer
    │
    └─ 3. manualExport(reportData, buffer)
          │
          ├─ 3a. PizZip 解压 → 读取 word/document.xml
          │
          ├─ 3b. mergeSplitRuns(xml)
          │       └─ 合并被 Word 拆分的 <w:r> 节点
          │
          ├─ 3c. 展开 {#algorithms}...{/algorithms} 表格循环
          │       └─ 复制 <w:tr> 行，替换行内占位符
          │
          ├─ 3d. 简单标签正则替换（reportPeriod 等）
          │       └─ problems 字段特殊处理：\n → <w:br/>
          │
          ├─ 3e. 写回 zip → 生成 Blob
          │
          └─ 3f. saveAs(blob, filename) → 触发浏览器下载
```

## 7. 关键实现细节

### 7.1 mergeSplitRuns — 合并 Word 拆分的 XML 节点

**问题**：Word 编辑 `.docx` 模板时，会按格式变化（加粗、字体切换等）将文本拆分到多个 `<w:r>` 运行（run）中。例如 `{reportPeriod}` 可能变成：

```xml
<w:r><w:rPr>...</w:rPr><w:t>{report</w:t></w:r>
<w:r><w:rPr>...</w:rPr><w:t>Period}</w:t></w:r>
```

正则 `\{reportPeriod\}` 匹配不到这种被拆分的文本。

**解决**：扫描所有 `<w:r>` 块，如果当前 run 的文本包含未闭合的 `{` 且下一个 run 的文本包含 `}`，则合并两个 run 的文本内容。

**注意**：当前模板经检测未出现此情况，但 `mergeSplitRuns` 作为防御性措施保留，以防未来模板被 Word 编辑后出现拆分。

### 7.2 处理顺序

```
mergeSplitRuns → 循环展开 → 简单替换
```

**不能颠倒**：因为 `{accuracy}` 在循环内外都有，先做简单替换会把循环内的也替换掉。

### 7.3 XML 转义

所有填入 XML 的文本值都经过 `escapeXml()` 处理：

```typescript
function escapeXml(s: string): string {
  return s
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
}
```

`{problems}` 字段特殊：换行符 `\n` 替换为 Word XML 的换行标签 `<w:br/>`（不转义）。

## 9. 如何对接真实数据

当前使用 `generateMockReport()` 生成 mock 数据。对接后端 API 时：

```typescript
// 1. 从 API 获取数据
const response = await fetch('/api/weekly-report')
const realData: WeeklyReportData = await response.json()

// 2. 传入 exportReport
exportReport(realData)
```

确保后端返回的数据结构符合 `WeeklyReportData` 接口即可。

## 10. 如何修改模板

1. 用 Word 打开 `public/templates/AI赋能周报模板.docx`
2. 编辑内容，保留 `{tag}` 占位符（花括号内不要有空格）
3. 保存为 `.docx` 格式
4. 如需新增字段，在 `WeeklyReportData` 接口和 `textReplacements` 对象中同步添加

> **注意**：保存后用 Word 重新打开确认占位符没有被 Word 自动拆分。如有拆分，代码中的 `mergeSplitRuns` 会自动处理。

## 11. 踩坑记录

| 问题 | 原因 | 解决 |
|------|------|------|
| 导出数据全部 undefined | `@click="exportReport"` 导致 Vue 把 MouseEvent 当作 data 参数传入 | 改为 `@click="exportReport()"` |
| docxtemplater 渲染失败 | 模板中 `{#algorithms}...{/algorithms}` 循环横跨表格单元格，引擎解析报错 | 移除 docxtemplater，改用纯 XML 正则替换 |
| `{accuracy}` 值错误 | 循环内外都有 `{accuracy}`，先做简单替换会把循环内的也替换掉 | 调整顺序：先展开循环，再做简单替换 |
| 占位符匹配不到 | Word 编辑模板时将 `{tag}` 拆分到多个 `<w:r>` 节点 | `mergeSplitRuns` 函数合并拆分节点 |

```

[周报] 模板加载完成, size: 1637366    ← 确认模板文件加载成功
[周报] 导出成功: AI赋能周报_xxx.docx  ← 确认导出完成
```

如需检查模板 XML 中的占位符，可在 Node.js 中运行：

```javascript
const PizZip = require('pizzip')
const fs = require('fs')
const buffer = fs.readFileSync('public/templates/AI赋能周报模板.docx')
const zip = new PizZip(buffer)
const xml = zip.file('word/document.xml').asText()
const tags = xml.match(/\{[^}]+\}/g)
console.log([...new Set(tags)])
```
